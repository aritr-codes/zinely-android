3. Claude implements.

# PR-A Step 5 Amendment

awaitReleased() release-completion contract.

Accepted after review. Codex approved (GO, no findings).

Commit:

ff3b9a1

Added:

* ProjectAutosaveHandle.awaitReleased(): suspend until the project id has left the registry

Contract:

* backed by a private, parentless CompletableJob completed by the factory AFTER unregister()
  runs in projectJob.invokeOnCompletion
* callers can only await the signal (no handle to complete or cancel it)
* awaitReleased() returning establishes a happens-before: the id is no longer mapped to this
  handle, so a subsequent create() for the same id is not rejected on account of this handle
* cancelling the awaiting coroutine cancels only the wait; the signal is untouched

This closes the reopen-ordering gap: re-create of the same project can be deterministically
sequenced after teardown completes.

EditorAutosaveBinder — lifecycle ownership.

Accepted after review. Codex approved (GO, after two requested lifecycle fixes applied).

Commit:

0a871a3

Implemented:

* EditorAutosaveBinder: owns one eager ProjectAutosaveHandle for the open-project lifetime
* ON_PAUSE primary flush
* ON_STOP backstop flush

* idempotent, memoized closeProject(): Job

* closed-state semantics: no resurrection (markDirty / lifecycle events inert after close)
* lifecycle + teardown flush failures routed to SaveFailureSink
* build.gradle.kts: added androidx.lifecycle.runtime.ktx (pure-JVM lifecycle-common types only)

Codex-requested fixes applied (both in commit 0a871a3):

* closeProject() now detaches the observer (closed = true → detachObserver() → launch teardown)
* observe() on a closed binder refuses to re-attach

Guarantees:

* teardown, once entered, always completes (NonCancellable) even if the autosave scope is cancelled
* the teardown flush is the final, non-concurrent write (binderMutex)
* closeProject().join() guarantees the project id has left the factory registry (reopen contract)
* a single handle owns the project id across lifecycle rotations (no duplicate coordinators)

Tests: 18 unit tests on the virtual clock via a hand-written FakeLifecycle (no Robolectric).

# Frozen Design Decisions

## Repository Consistency Model

Source of truth:

document.json

Tier A metadata:

* format
* paperSize
* schema version

Derivable.

Tier B metadata:

* projectId
* updatedAt

Derivable.

Tier C metadata:

* title
* createdAt

Not derivable.

Future metadata ADR required.

No universal rebuildability claim exists.

---

## Cancellation Model

Cancellation may occur:

* before commit
* after commit
* during coroutine resumption

Cancellation must not interrupt an entered atomic commit.

Rules:

* no runInterruptible around commit
* no suspend work inside commit
* AtomicFileStore remains synchronous
* teardown flush uses NonCancellable (Step 6)

---

## Path Safety Model

Project IDs are untrusted.

Requirements:

* whitelist validation
* normalize()
* startsWith(projectRoot)
* single chokepoint
* no ad-hoc path construction

Repository path:

filesDir/projects/<projectId>/document.json

---

## Save Failure Model

Failure sink:

* in-memory only
* per-project keyed
* latest failure retained
* failureCount since last clear
* explicit clear
* no persistence

Background autosave failures flow into the sink.

Foreground flush failures are returned directly and not sink-reported.

# PR-A Step 5 — Design Record

Status:

Implemented and accepted (see "PR-A Step 5" and "PR-A Step 5 Amendment" above; commits c3051c0
and ff3b9a1). The originally approved design is retained below for history.

---

## Step 5 Goal

Introduce:

AutosaveCoordinatorFactory

Responsibilities:

* Construct AutosaveCoordinator instances.
* Wire coordinator failures into SaveFailureSink.
* Bind coordinator saves to DocumentRepository.
* Enforce one active coordinator per project.
* Manage project-level coordinator ownership.

No Android lifecycle integration.

No Hilt.

No UI wiring.

---

## Approved Step 5 Design

Factory owns:

* coordinator creation
* failure collector creation
* project lifetime

Factory must enforce:

One active coordinator per project.

Duplicate creation for the same project must fail fast.

Factory maintains:

```kotlin
active[projectId] -> ProjectAutosaveHandle
```

with synchronized registration and removal.

---

### Handle API

```kotlin
interface ProjectAutosaveHandle {
    fun markDirty()

    suspend fun flushNow(): DataResult<Unit>

    suspend fun close(): DataResult<Unit>

    fun cancel()
}
```

Meaning:

* flushNow() = immediate save
* close() = flush then cancel
* cancel() = hard teardown

The Step 5 amendment (commit ff3b9a1) later added `suspend fun awaitReleased()` to this interface
(see "PR-A Step 5 Amendment" above).

NonCancellable orchestration was deferred to Step 6 and is now implemented in EditorAutosaveBinder
(commit 0a871a3).

---

### Registry Requirements

* Duplicate create(projectId) rejected.
* cancel()/close() unregister project.
* Parent autosave scope cancellation unregisters project.
* Identity-safe unregister.
* Single-writer invariant enforced.

This design was approved after Codex identified and closed a duplicate-coordinator durability risk.
